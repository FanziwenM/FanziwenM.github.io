PYTHON在UE4中的运用

Describes how to use Python in the Unreal Editor to script content production tasks.

可以通过Unreal Editor中的Python做到：

  • 构建跟大体量的字长管理或工作流。更好的跟其他的3D应用程序联系起来
  • 在UE4中自动执行耗费时间的管理认为，例如，批量的自动生成LOD
  • 程序化的将内容中的资产布置到关卡中
  • 通过Python创建UI在UE4编辑器中进行控制

UE4的Python版本为2.7，如果想使用不同版本的Python 可以通过环境变量UE_PYTHON_DIR，来重新指定安装在本地的，你所希望嵌入的Python版本，然后重新编译引擎

和蓝图(Blueprints)不同，Python的执行环境只在编辑器中有效，在项目运行的时候则无效。
意思就是，可以用Python自由的在编辑器中编写自动化的脚本，编辑或者创建资产。但是不能用Python作为一个编写游戏逻辑的语言。

用法

Ways to Run Python Code in Unreal Editor

这里有几种不同的方法在编辑器中运行Python脚本，每种方法略有不同，可以选择适合自己的方法

  • Python Console:在Output Log中切换成Python，就可以和Python自带的交互式shell一样，可以逐句的执行。如果需要多行则要使用Shift+Enter
  • Cmd Console:在Output Log中，即使在Cmd下，也可以用py开头，来执行Python语句或文件
    py "C:\MyScripts\myScript.py"
    py print "Hello World"
  • The File Menu: 通过File/RecentPythonScripts来执行最近执行过的Python脚本

API参考

这一节将介绍在使用Unreal Python API时要注意的一些事情

资产

Assets
如果需要对项目中的资产(assets)进行操作，要用Unreal Python API。不要直接用文件管理模块(例如os)在磁盘上对本地的项目文件进行操作。
然而，建议使用unreal.EditorAssetLibrary模块，或者使用unreal.AssetTools类。

属性编辑

在Unreal的Python API 参考中，你会发现每个类中描述下都列出Editor Properties(可编辑属性)而这些属性是可以通过函数get_editor_property()和set_editor_property()进行读写。

这些类的属性可以像正常的Unreal的可编辑属性(Editor Properties)用set_editor_property函数来编辑。也可以像一个正常的对象属性(object property)直接设置。
但是get_editor_property()函数可以访问一些没有直接在暴露成对象属性的属性。

当一个属性可以同时以可编辑属性(Editor Properties)和对象属性(object property)进行访问编辑时。最好用函数set_editor_property()。
因为用函数set_editor_property()进行参数修改时，编辑器界面(Editor UI)会同步进行修改，尤其是Details面板里的那些修改。
