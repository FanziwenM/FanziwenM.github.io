  • 在Grid和Sphere后面添加Color

  • 然后在Attributes中新加一个属性Cd

最后移动球体看看效果：

Attribute Transfer最常用的参数：

  • Distance Threshold: 传递范围阈值，既目标模型距离源模型距离多少以内属性会被传递；
  • Blend Width: 混合宽度，既传递属性的值于原值进行混合过渡的宽度；

我们可以通过Marker来看看下图中属性temp的传递效果

Sphere的属性temp的值为1，而Grid的值为0，传递后0和1中间出现的小数就是它们的混合值，而Blend Width就是调整0到1的过渡宽度

📌Tips

Intrinsic

固有属性

在模型的Primitives下可以看到很多隐藏的固有属性(Intrinsic)
例如在使用节点Copy to Points并勾选Pack and Instance的情况下。Intrinsics就会多出transform属性，一个3x3矩阵。

然后可以通过读取或设置transform矩阵来改变模型信息，
使用到VEX函数:

  • primintrinsic()
  • setprimmintrinsic()

Digital Asset

虽然Digital Asset可以存储在同一个Digital Asset Library(.hda文件)里。但是官方还是建议每一个Digital Asset存在一个单独的文件里，方便查找和分享。

However, we recommend you store each asset in a separate ibrary file. This makes it straightforward to share some assets and not others, and keep track of which assets are in which files.

ParameterInterface

MenuParameter

菜单栏参数

整型(Integer)，字符串(String)，文件(File)，节点路径(Node Path)等类型的参数，是可以设置成下来菜单

例如：

  • 创建一个整型参数
  • 选择新创建参数，点击 Parameter Description 中的 Menu 标签
  • 在 Menu Items中创建需要的键值
  • 最后勾选Use Menu Item Token as Value

在整型的参数(integer parameter)中，默认设置是用户选择一个菜单选项，返回的值时这个选在在这个菜单中的索引(Index)。例如，选择第一个菜单项，返回值为0，第二个返回值为1,等等…比如上图中，Label 是 six虽然对应的值是6，但是默认会返回2
所以要勾选Use Menu Item Token as Value，这样返回的值就是Token中设置的值，选择six就会返回6

如果Token的值不是整型，返回的值也是菜单项的索引(Index)

👿 Trick

选取整数点

对一堆点云进行取整，可以直接用对点位置@P进行四舍五入rint()

可以在四舍五入前除以某个数，并在之后再乘回这个数进行类似隔断取整的效果：

通过上述办法只是将点位置取整，点数还是不变，所以最后还是要通过节点fuse合并重合点。

同样的处理方法还可以处理通过VDB转换出来的高面数的Polygons，还是一个整体模型的情况下，让模型点数变得可控，并且横平竖直。但缺点也是因为这样的布点，会丢失弧面细节。

这样的方法可以很好的处理程序化建筑的墙体，因为它需要的是一个简洁一体的，布线垂直的模型。

DisplacementTrick

Displace基本都是通过Normal改变模型，例如节点mountain。
我们会经常遇到下面这种情况：

因为Noise的强度太大而产生的面穿插。但是更多情况是因为模型的部分太薄。
然而，这种情况可以通过改变其Normal修复：

  • 先通过VDB 将模型转变成圆滑效果不错的模型，主要目的是为了圆滑类似耳朵这样的比较薄的部分。
    然后将模型的Normal传递给源模型：
  • 之前提到，Displace是基于Normal改变模型，所以这一步的目的就相当于简化并Smooth了下Normal
  • 而这样Displace出来的模型，在保证基本一样的效果同时，也避免了面穿插
