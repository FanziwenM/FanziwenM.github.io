📦Package

Python包

随着程序变得越来越长，你或许会想把它拆分成几个文件，以方便维护。你亦或想在不同的程序中使用一个便捷的函数， 而不必把这个函数复制到每一个程序中去。
为支持这些Python有一种方法可以把定义放在一个文件里，并在脚本或解释器的交互式实例中使用它们。这样的文件被称作模块(Module) ；模块(Module)中的定义可以导入到其它模块(Module)或者主(Main)模块

Name

全局变量__name__

每一个.py文件都可以视为一个模块(Module)。而且当一个模块被导入时，可以通过全局变量__name__获取名字(字符串)。

模块可以包含可执行的语句以及函数定义。这些语句用于初始化模块。它们仅在模块第一次在import语句中被导入时才执行。(当文件被当作脚本运行时，它们也会执行。)

正如上面所说的模块被导入时，其中的执行语句也会跟着一起运行，但是我们经常导入一个模块(Module)只是为了引用其中的某些函数(Function),这个时候我们就需要在模块(Module)中做一些检测。

全局变量__name__还经常用于检测模块(Module)是被引用还是被执行，如果是被直接执行的话则__name__的值为字符串"__main__"

import

导入的更多信息

模块可以导入其它模块。习惯上但不要求把所有import语句放在模块（或脚本）的开头

每个模块都有它自己的私有符号表(private symbol table)，该表用作模块中定义的所有函数的全局符号表。因此，模块的作者可以在模块内使用全局变量，而不必担心与用户的全局变量发生意外冲突。modname.itemname

import语句有一个变体，它可以把名字从一个被调模块内直接导入到现模块的符号表里
