初始阶段

  • Awake（唤醒）
    • Awake在Start前（所有物体的Awake执行完之后执行Start）
    • Awake在GameObject创建后就执行
  • Start（开始）
    • Start在GameObject创建后并且Scripts启用后才执行
  • OnEnable（启用）
    • Scripts启用就调用（多次）

物理阶段

  • FixedUpdate（固定更新）
    • 适用物理操作（移动，旋转…）,不受FPS影响
  • Update（帧更新）
    • 处理游戏逻辑
  • LateUpdate（延迟更新）
    • 跟着Update后更新（同一帧），适用于跟随逻辑

输入事件

  • 鼠标事件
    • OnMouseDown
    • OnMouseUp
    • OnMouseEnter
    • OnMouseOver
    • OnMouseExit

渲染阶段

需要有 Renderer 组件

  • OnBecameVisible
    • MeshRenderer可见调用
  • OnBecameInvisible
    • MeshRenderer不可见调用
  • OnPreCull
    • 在摄像机执行Cull前执行代码.
  • OnRenderImage
    • 在摄像机结束渲染后调用，可以修改最后呈现的效果。

自定义编辑器

Custom Editors

[ExecuteInEditMode]
当创建一个继承于MonoBehaviour的脚本时，我们可以通过[ExecuteInEditMode]使其可以在编辑模式(edit-time)下正常执行。但是它并不算是一个编辑器工具

一个自定义的编辑器，是不同于默认的C#脚本格式：

  1. 创建一个C#脚本，命名为{MyScriptName}Editor(ScriptName为你希望自定义其编辑器的脚本。；
  2. 类要继承与 Editor (需要使用命名空间using UnityEditor)；
  3. 在类前添加属性CustomEditor: 告诉unity哪个组件的编辑器需要被改变；

  4. 在类前添加属性CanEditMultipleObjects： 告诉unity可以多选物体同时修改；

  5. 通过重写OnInspectorGUI()函数，来修改Inspector界面

事实上Unity中监测面板上看到的所有类型的组件，都可以在这里进行自定义，包括一些复杂的设置，地形系统，动画导入设置等等。

渲染

环境

Content and Context

The Context

上下文

Unity将渲染分为照明(Lit)和渲染(Rendered)两部分，而Context就是指场景中已经存在的，那些照明物体的灯光。

默认的灯光环境
当新建一个场景的时候就会有个默认的灯光上下文(Default Lighting Context)，一个天空球，直射光。

  • 天空盒(Skyboxes)
    天空盒（烘焙或程序化）可作为光照设置的组成部分。除渲染天空外，天空盒还可用于控制环境光照(ambient)和物体反射(reflections)。
  • 全局光照(Global Illumination)
    全局光照是Unity的重要概念。标准着色器(Standard shader)和Unity的GI系统都经过精心设计，可以很好地相互配合。
    GI系统负责创建和跟踪反射光、发光材质发出的光以及环境光。

The Content

内容

内容(Content)是用于描述场景中正在渲染的对象的术语。它们最终呈现的效果，是光照上下文(Context)结合运用在物体上的材质(materials)的结果。

常用结构体

Universal Render Pipeline

普通渲染管线

使用URP

  1. 在创建工程的时候直接选择Universal Render Pipeline工程
  2. 在Window>Package Manager中直接添加Universal RP包

如果通过Package Manager添加URP，在使用URP前要先修改渲染设置：

  1. 在资源窗口创建一个Universal Render Pipeline Asset；
  2. 在Edit>Project Setting…>Graphcis中的ScriptableRenderPipelineSettings里添加刚刚创建的URP资源

Universal Render Pipeline Asset

UPR渲染设置

URP资源（URP Asset）可以控制URP的一些渲染特性和质量。它是继承与RenderPiplineAsset的可编写对象。
你可以拥有多个URP Asset，这样可以很方便的进行渲染设置的切换。但是不能在HDRP/SRP和UPR直接进行切换。

Tips

  1. 调试：
    1. 控制台
    2. 用共有变量通过面板检测数值
    3. Inspector>Debug Mode， 观察更多参数
  2. Edit>Preferences>Colors>Playmode tint，预防在rumtime的时候进行场景编辑；
  3. Ctrl+Shift+F，将选择物体移动到当前视角；
  4. [Header(String)], [Space]， 参数面板的排版；
  5. Copy Component, 可以在Runtime的时候调整合适的参数，然后拷贝为默认值。

其他

  • 网格(Meshes)主要的渲染图元(Primitive)，它们定义了一个物体的形状(Shape)

  • 材质(Materials)决定了一个模型的表面以何种形式进行渲染，包括要用的的贴图(Textures)，贴图的重复(Tiling)信息，颜色等各种各样的属性。而一个材质有多少属性可以进行调整，则由着色器(Shader)来决定。

  • 着色器(Shader)是一小段代码，包含了一些数学计算和算法。可以通过上面提到材质(Materials)给到的一些属性(还包括灯关，环境等其他参数)，计算出各种各样的显示效果。

  • 贴图(Textures)是一张位图(bitmap),一个材质可以引用一张或多张贴图。在进行着色的时候，可以采样贴图来添加更丰富的颜色到物体表面(贴图可以表示除颜色以外如粗糙度，金属度等其他信息)。
