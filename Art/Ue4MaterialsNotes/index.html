UE4材质基础

发表于 2019-06-02 更新于 2019-08-15 分类于 Art

🌏基础介绍

材质是用了一种特别的编程语言：高阶着色器语言(High Level Shading Language)简称HLSL 创建的。HLSL可以使材质直接和图形硬件通信，从而使美术和程序员直观的观察控制屏幕显示内容。Unreal Engine 4 (UE4)中用来创建材质球的材质节点包含着和HLSL一样的代码块，来实现和这些代码块相同的效果。

主材质节点(The Main Material Node)

在Content Browser中右键创建一个材质

双击我们新建的材质，进入材质编辑窗口。

主材质节点(The Main Material Node)可以被认为材质制作的最后一步。将材质节点任意组合后，输入给主节点，就会有相应的效果在材质球编译后显示。

主材质节点的每一个输入(Input)都有其特殊的效果。白色的输入处于启用状态，说明这类材质有相应的属性可以输入，而灰色的输入处于禁用状态，说明这个材质没有这些相应属性，而且不能输入信息。想启动和禁用主材质节点的输入，就要修改Details面板材质球的属性。

Details面板 会显示当前选择的节点的信息,所以查看主材质的信息时请选中主材质节点，或者左键单机空白区域。

经常使用，进行修改的地方：

  • Blend Mode(混合模式)
  • Shading Model(着色模式)
  • D3D11Tessellation Mode(曲面细分模式)

例如，当我们需要一个透明材质，我们就需要修改Blend Mode(混合模式)，改为Translucent,我们就会发现主材质节点中之前灰色的输入Opacity变成了白色开启状态。并且关闭了一些不适用这个材质的输入。

当我们需要一个无光材质，我们就需要修改Shading Model(着色模式)，改为Unlit.主材质节点就只剩下Emissive Color等相应的输入

有些材质会对面数的要求比较高，要用到高度或置换贴图，这个时候就需要修改D3D11Tessellation Mode(曲面细分模式)

其他的材质设置当后面用到相应的材质时会详细介绍。

节点的使用(Usage)

我们可以再空白的地方右键，或者在右边的Palette面板中搜索要使用的节点，比如想使用一个常量。右键然后键入constant

很多常用的节点都有快捷键，就比如Constant的快捷键是1
所以我们可以按住1然后再空白处鼠标左键，同样可以呼出Constant节点

选中我们刚刚呼出的节点，Details面板中就会显示该节点的设置信息。比如Constant节点，其中Value是给我们设置该常量的值。

创建三个Constant，然后分别链接给Base Color,Metallic, Roughness。将Base Color连接的常量设置成1,Metallic设置成1, Roughness设置成0。Ctrl+S储存。这样，我们就创建了一个最简单的材质，光滑的金属材质：

实例化材质(Material Instance)

以上面我们刚刚做的金属材质Metal为例，找到我们刚刚做的材质球，右键选择Creat Material Instance

我们会看到我们多了一个材质球Metal_Inst

我们会发现Metal_Inst下面的绿色会比较暗，而且如果你鼠标悬停在这两个材质球上面，你会发下它们的类型也不一样。一个是Material，一个是Material Instance，最重要的是，当你双击进入材质球编辑，他们的界面完全不一样

但是，我们发现虽然类型不一样，界面也不一样，但是它们的效果似乎一模一样。所以为什么要多一个Instance类呢？
这有点像编程中类(Class)的概念

类(Class)

类是对现实生活中一类具有共同特征的事物的抽象。如果一个程序里提供的类型与应用中的概念有直接的对应，这个程序就会更容易理解，也更容易修改。类有三大特性:封装性,继承性,多态性

区别类(Classs)，对象(Object)，实例(Instance)
上面的小明就叫作对象(Object),但也可以说小明是人类这个类(Class)的实例(Instance)

但是这还不足以让我们创建出丰富多彩的人类，就好比小红是女的，小明戴眼镜，小文一米八等等，世界上的人又不是都一个样。所以这就要讲到参数(Parameter)

这样，我们就创建出了三个不同的人，而且我们还能通过我们定义的人类这个类(Class)创建出更多的人。
最重要的还是，因为继承性，假如我们需要为我们的人类添加更多属性，比如我们忘了给小明，小红，小文安两个耳朵，我们只要在人类这个类(Class)中在加上属性两个耳朵，这样小明，小红，小文等只要是通过人类创建出来的对象(Object)就都多了两个耳朵。
当然，为了创建更丰富的人，我们就要在人类中添加更多的参数，比如体重，头发颜色，皮肤颜色…等等，让他们在这些属性上存在差异，就会丰富我们的对象(Object)

之前提到，类(Class)是一个抽象的概念，所以类是不能直接拿来用的，就好像人类不是一个实体。类似类的Material其实也一样，不能直接赋予给模型。虽然UE4里可以直接把Material赋给模型，但是为了方便实时查看效果，也为了方便复用，而且也比较符合类的概念，所以建议创建Material Instance再赋予模型。

一般母材质(Material)都是以M_为前缀，，而实例化材质(Material Instance)则MI_为前缀的。不仅UE4官方，很多项目都是这样命名。

参数 (Parameter)

上面在实例化材质中已经提到过参数(Parameter)了。在这里，我们将了解跟多Material中的参数和如何使用它们。

  • 在材质编辑器中的空白处右键，键入Parameters就可以看到相关节点。
  • 但是一般情况下还是像之前提到过的右键换为参数：

我们经常需要转成参数节点：Constant,Constant3Vector,TextureSample
和它们转换为参数状态的节点：ScalarParmeter,VectorParameter,TextureSampleParameter2D

我们知道Constant的快捷键是1，Constant3Vector的快捷键是3。
而他们的参数节点ScalarParmeter的快捷键是S，VectorParameter的快捷键是V。
所以右键Constant转成参数，和按住键盘S加鼠标左键呼出的ScalarParmeter节点是一样的。

材质函数(Material Function)

材质函数(Material Function)可以使我们做的材质里的节点更加简洁，而且使我们在制作材质的时候更加方便。

在编程里面，如果出现一大段重复的代码，对于程序员来说是一件非常Low的行为。所以程序员们会把要经常反复使用的代码写成一个函数(Function)，然后开放几个需要修改的参数(Parameters),通过调用函数(Function)实现相同的功能。不仅编程过程更加方便，还可以减少大段重复的代码，使代码更加简洁。

所以养成使用材质函数(Function)是一个很好的习惯。
在使用材质函数(Material Function)前，我们要自己创建一个材质函数(Material Function)，和创建材质(Material)一样在Content Browser下右键创建一个材质函数(Material Function)

双击打开我们新建的材质函数(Material Function)，我们会发现它的编辑界面和材质(Material)几乎一样(可使用的节点也基本一样)，但是材质函数(Material Function)没有主材质节点，不过多了一个Outp Result

我们知道材质(Material)的主材质节点只有一个，但是材质函数(Material Function)的Outp Result可以不止一个，但至少要有一个。我们可以复制节点Outp Result，也可以右键键入FunctionOutput。

记得之前我们做的M_Standard材质中有一个调节Normal强度的功能，我们可以试着把这部分功能做成一个材质函数(Material Function)。
我们直接找到之前那部分功能，框选这部分功能，然后Ctrl+C复制

然后到我们新建的材质函数(Material Function)里，Ctrl+V张贴

然后我们开始分析，我们要做的材质函数(Material Function)要有几个输入(Input)几个输出(Output)
我们既然要控制Normal的强度，我们肯定要输入一张Normal，然后我们肯定还要输入一个数来调整强度，最后就是把结果输出，所以我们要两个输入(Input)和一个输出(Output),右键键入Input就能找到节点FunctionInput

  • 假设A为向量（三维数组），而B为标量，则A数组中的数逐个与B进行运算，例如乘法：

对于对数字不太敏感的美术，或者就想知道用这些基础运算的节点到底有什么用，使用单个节点有什么效果，可以自己进行尝试，加深映象。

Append

添加，附加

在编程里Append是一个对列表(list)的基础操作。作用就是将一个元素(items)添加到一个列表内。
例如我有个一个列表里面有三个数[2, 3, 1]假如我要加一个数0.5进去，如果是用之前上面说的Add，结果可想而知，肯定是[2.5, 3.5, 1.5]，其实还是一个有三个元素的列表。如果是用Append，结果就是[2, 3, 1, 0.5]，变成了一个有四个元素的列表。

其实RGB图也可以看成一个列表(list)时,其中的R,G,B就相当于它的元素(items)。只不过作为美术，我们更习惯说R通道(channel)，而不是说元素(items)。所以当我们看到一组数来表示颜色时,比如[1, 0, 0]就是代表红色，因为这组数组表示的是R=1,G=0,B=0。就比如Constant3Vector节点当值为[1, 0, 0]它的颜色时红色。

Inputs Description
A 输入将被添加到的值
B 输入要添加的值

我们知道一组数[1, 0, 0]代表红色，那我们可以试着用Append做一组数来代表天蓝色，原理很简单，我们将0.2添加一个0.4,这时候我们得到一组数[0.2, 0.4]，再将[0.2, 0.4]添加一个1,这时候我们就得到一组数[0.2, 0.4, 1]。

一定要注意Append的顺序，如果要用一组数来表示颜色，列表第一个数一定是代表红色R,第二个数代表绿色G，第三个蓝色B，如果你添加了第四个数，则代表透明通道A

例如下图，在将[0.2, 0.4]添加一个1的时候我把Append输入A和B颠倒，所以表示的意思是:将1添加一组[0.2, 0.4]。
所以最后的结果就是[1, 0.2, 0.4]，自然也就变成了下图的颜色。

Lerp

线性差值(LinearInterpolate)

线性差值Lerp节点通过第三个值(Alpha)讲两个值(A&B)进行混合。如果Alpha的值是0返回A的值，如果Alpha的值是1则返回B的值。如果Alpha在0和1之间，比如0.5则AB取平均值混合。

Inputs Description
A 当Alpha为0时要输出的值
B 当Alpha为1时要输出的值
Alpha 作为控制AB混合的值可以充当Mask

作为美术有个非常直白的理解，就当A是PS里一张底图，而B则是一张新建图层，而Alpha就是新建图层的遮罩(Mask),跟所有Mask原理一样:黑透白不透

ComponentMask

部分提取，快捷键Shift+C

经过Append的介绍，我们大致对列表(list)有初步的了解后，我们就可以认识ComponentMask。这个节点方便我们提取一个列表(list)内特定的元素。对于一张RGB图来说，我们更习惯说提取一张图的某个通道(channel)。而Append刚好也符合我们美术的习惯。

Properties Description
R 如果勾选，红通道(第一个元素)将会输出
G 如果勾选，绿通道(第二个元素)将会输出
B 如果勾选，蓝通道(第三个元素)将会输出
A 如果勾选，透明通道(第四个元素)将会输出

StaticSwitchParameter

静态开关参数

一个布尔型(Boolen)型参数,输入两个值，当Default Value勾选输出第一个值，反之则输出第二个值

Inputs Description
True 当Default Value勾选将输出的值
False 当Default Value未勾选将输出的值

做一个简单的例子，这边使用一个红色[1, 0, 0]与StaticSwitchParameter的True相连绿色[0, 1, 0]则和False相连。选择节点StaticSwitchParameter命名为UseRed,检查Detail面板上的Default Value为未勾选状态。

然后创建一个该材质的Instance，就会看到参数面板上有一个UseRed参数，当我们将参数勾选，就会发现材质球变成了红色

Material Instance的参数(Parameter)，前面的勾选框是开启使用这个参数，后面才是真正对参数进行修改的值(Values)。前面的勾选框如果不勾选则使用我们之前定义好的默认值。在之前对Material Instance调整参数时大家应该会注意这一点，但是使用StaticSwitchParameter作为参数时，不要搞混前后两个勾选框的作用

Clamp

限制到固定范围

Clamp节点会将输入的值，通过设置最大值和最小值限制到规定的范围内。

Details Description
Clamp Mode 设置限制模式，可以只限制最大值或者最小值
Min Default 设置最小值的默认值
Max Default 设置最大值的默认值
Inputs Description
Min 输入将要限制的最小值
Max 输入将要限制的最大值

我们可以再一次利用Debug节点DebugFloat3Values做个简单的检测，如下图：

我们用Constant3Vector设置为[-1, 1.5, 0.8]，输入Clamp将值限制到最小为0,最大为1之间。所以-1变成了0,1.5变成了1.0, 但是0.8因为在限制范围内，所以还是0.8。所以最后的结果就是Debug节点显示的结果：[0.0, 1.0, 0.8]

我们会经常在很多材质里看到别人经常使用Clamp，因为很多输入比如Roughness的取值是[0-1],这是一个很好的习惯，而且合理的限制方便之后的运算。

绝对世界位置

节点AbsoluteWorldPosition可以获取Shader上每个像素的位置信息

预览窗口的Cube长宽高貌似为256个单位，所以相对于Cube的中心点(世界(0,0,0)坐标)，顶点坐标为(128, 128, 128) (128, 128, -128) (128, -128, -128)...等八个位置。所以如果直接右键预览AbsoluteWorldPosition或者连接在主材质节点的EmissiveColor上会出现爆亮的效果。

因为UE4的RGB取值在0到1如果小于等于0则为黑，大于1则会发光，数值越大，越亮。

MaterialAttributes

材质属性

很常用的一类节点，一个材质里可能有多个子材质，或者需要用到某个材质的某个材质属性。而MaterialAttribute的一系列方法，可以通过Make，Break，Get和Set等方法创建，拆分，获取和设置一个材质。

例如和创建Material一样连接节点MakeMaterialAttributes：

SetMaterialAttributes

设置材质属性

节点SetMaterialAttributes和节点MakeMaterialAttributes很像，都是要输入一些材质属性创建一个材质。不同的是MakeMaterialAttributes会暴露出所有材质属性，而SetMaterialAttributes要自己添加希望设置的材质属性。

和MakeMaterialAttributes一样SetMaterialAttributes只有一个输出，而且默认情况下只有一个输入MaterialAttributes。

材质的单个属性默认是全部关闭的，假如我要添加某个材质属性，则需要在Details面板中的Attribute Set Types通过点击加号添加

当添加新的单个属性时，默认给我们添加了BaseColor

📋其他节点

Debug节点的使用

Debug节点可以以数值的形式显示结果，这样可以帮助我们在完成一些计算后检查输出结果是否正确。

  • 右键，键入Debug，就可以看到相关节点

要注意的是，你要检查的输出值是什么类型就要用什么类型的Debug节点：

  • DebugScalarValues检查一维数(标量)
  • DebugFloat2Values检查二维数组(2D向量)
  • DebugFloat3Values检查三维数组(3D向量)

还有其他Debug节点（不常用）：

  • DebugBinaryValues是以二进制的形式显示输出。明显就是给大佬们用的
  • DebugOnOff不断循环的输出0和1，也就是True和False
  • DebugTimeSine输出一个0到1的循环
