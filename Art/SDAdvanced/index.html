节点解析

VectorMorph(Warp)

根据向量变形(弯曲)

我们知道DirectionalWarp可以通过使用一个Mask对目标图案进行弯曲(Warp)。但是该节点只能决定时候弯曲(Warp)(mask为1处弯曲，0处不弯曲)。所以DirectionalWarp带有很强的方向性(Intensity越强方向性越强)，所以会造成很明显的偏移效果:

如上图所示通过 DirectionalWarp 原图很明显的往左偏移，如果我们希望保持原图的大致位置，就需要可控性更高的 VectorMorph 节点

为什么说 VectorMorph 的可控性更高。
因为 VectorMorph 是通过VectorField(向量场)来进行变形。而VectorField则是通过一个Color图来表示。

有图形学基础的人一定会知道，一张Color贴图上的像素的RGB值可以表示为一个向量(Vector)的XYZ，典型的例子就是法线贴图(Normal)。法线贴图就是用颜色来描述像素在模型表面的法线朝向。
但是向量的XY是存在负数值的，贴图的RGB是没有负数值的，基本都是0.0-1.0或0-255。
所以，一般在三维空间表示单位向量的区间[-1,1]被压缩到了区间[0,1](公式也很简单(x+1)/2)
压缩后，原本的0就变成了0.5。例如向量(0,0,1)就变成了(0.5,0.5,1)

回到节点VectorMorph的VectorField。原理是一样的，只不过因为是二维空间，所以只用到XY的值也就是RG的值。
所以在二维空间中，想创建一个朝左的向量它的XY就是(-1,0),转换成颜色RG就是(0.0, 0.5),朝右的向量它的XY就是(1,0),转换成颜色RG就是(1.0, 0.5)。然后加上无关紧要的B的值，我们就得到了两个颜色：

  • Color1:(0.0, 0.5, 0.0)
  • Color2:(1.0, 0.5, 0.0)

而节点VectorMorph就可以利用这两种颜色将原图一部分往左变形，一部分往右变形：

如果你真的理解了它的原理，你可以让它以任何方向变形。再比如上下：

最后，我们回到最初遇到的问题：“如何在保持位置不发生偏移的情况下Warp”：

还有个类似的节点VectorWarp原理和VectorMorph一样，也是通过VectorField来进行处理。只不过是扭曲(非线性变换)。
而VectorWarp和Non Uniform Directional Warp的效果和作用比较类似。

正如我们在上面提到的法线贴图(Normal)也是用像素颜色来表现向量的值，所以我们自然可以用Normal当输入给VectorField

TileSampler

重复采样器

非常常用的用来制作重复纹理的节点。相比节点TileRandom和TileGenerator。它的多了很多辅助贴图的输入 ，而这里主要讲解这几个贴图输入的作用。

ScaleMapInput

缩放贴图，输入一张灰度图(Grayscale)，通过属性菜单中的Size|Scale Map Mutiplier影响元素缩放的强度

DisplacementMapInput

位移贴图，输入一张灰度图(Grayscale)，通过属性菜单中的Position|Displacement Map Intensity来调整偏移强度，而它下面的Displacement Angle则是调整偏移方向

RotationMapInput

旋转贴图，输入一张灰度图(Grayscale),通过属性菜单中的Rotation|Rotation Map Multiplier来控制旋转强度，重复的元素就会根据输入的灰度图，进行不同程度的旋转

VectorMapInput

向量贴图，输入一张彩色图(Color)，跟上面讲到的节点VectorMorph的原理一样，要根据一张彩色图(即VectorField(向量场))。来进行可控的偏移，和旋转。Position|Vector Map Displacement则控制偏移强度，Rotation|Vector Map Displacement则控制旋转强度

效果和输入DisplacementMapInput类似。区别就是在于DisplacementMapInput只能输入一张灰度图，只能定义一个方向，进行偏移或不偏移。VectorMapInput则可以通过一个色彩图，定义两个方向，进行两个方向的偏移。例如上图就是定义了两个2D向量[0.0, 0.5] 和[1.0, 0.5]即向左，和向右

同样，也可以直接输入Normal:

ColorMapInput

颜色贴图，输入一张灰度图(Grayscale),通过属性菜单中的Color|Color parameterization Multiplier来控制颜色(准确的说是亮度)

重复元素的亮暗，除了追常用的随机，和上面介绍的用输入的ColorMap来控制。还有比较常用的就是通过大小来控制颜色，在 Color Parametrization Mode中选择Scale

效果：

MaskMapInput

遮罩贴图，输入一张灰度图(Grayscale)，控制重复元素是否显示。通过Mask Map Threshold来控制遮罩的阈值

PatternDistributionInput

分布贴图，输入一张灰度图(Grayscale)，控制输入图案的分布

Pattern Distribution仅在多个图案是输入PatternInput时起作用，用于分配多个图案的分布情况

在了解了TileSampler其他的输入后，就可以利用这些输入制作以下案例：

  • 石头的散落
  • 草的散落
  • 花蕊散落

NonUniformDirectionalWarp

不均匀带方向性弯曲

之前提到的节点VectorMorph有很高的可控性。但是每次都要将一张灰度图(Grayscale)转成一个彩色图(Color)。还是有些繁琐。
但是对弯曲方向性的要求不是很高，其实就可以用到节点NonUniformDirectionalWarp:

我们可以看到上图使用两种方式将原图进行Warp的效果其实类似（除了扭曲方向不太一样）而用NonUniformDirectionalWarp可以通过两个灰度图输入(Grayscale)来控制不均匀的强度(Intensity)和角度(Angle)

使用NonUniformDirectionalWarp可以方便快速进行非均匀的弯曲(Warp)，它可以做出很多漂亮的Noise，例如：

  • 地衣噪点
  • 其他脏迹0

SlopeBlur

坡度模糊

SlopeBlur非常常用，通过输入的图(高度图)的坡度(Slope)来进行带方向性的模糊(Directional Blur)。所以内部逻辑和节点Directional Warp相似。
众所周知，最经常的用法就是通过一个Noise来打破直线，也常用来做边界磨损效果。或脏脏迹、污垢和铁锈。

SlopeBlur的基本用法在这不再赘述，要注意的就是模式(Mode)的不同效果。

然而SlopeBlur还有个很好用的效果就是膨胀(inflate)。
将原图与自身SlopeBlur可以达到很好的膨胀效果，用这种方法可以很好的消除高度图上太过锐利的边界:

通过SlopeBlur模糊自身的其他应用：

  • 地衣噪点
  • 苔藓噪点0
  • 苔藓噪点1

Gradient(Dynamic)

渐变(动态)

一种重新映射的Gradient，在某些地方可以替换普通的Gradient节点。
因为普通的Gradient节点的颜色的定义是在节点内部，而且不能转换成参数(ExposingParameter)。所以如果需要将颜色定义成参数，就可以用新建一个颜色输入，输入给Gradient(Dynamic)的GradientInput

Gradient(Dynamic)最主要的控制是GradientInput输入的色彩图(color)。可以根据映射方向(Orientation)，和通过滑块(slider)控制映射位置的偏移。这说明，我们可以将不同方向的多种渐变的颜色存在一张图中。通过滑块来切换不同的渐变模式

例如：

  • 像第一张图一样，先创建一张随机颜色的贴图，输入给GradientInput
  • 在节点Gradient(Dynamic)中的Orientation选择水平Horizontal,然后Position调整到0.5
  • 这样效果就相当于提取了输入的色彩图，水平的，中间位置的，颜色信息

参数化

Exposing Parameter

参数的显示/关闭

Display/Hide inputs and outpust with conditions

当我们暴露出一些参数的时候,有些参数直接可能有关联：有些参数可能是基于另一个参数的状态。

可以通过使用Visible if来添加显示参数的条件(甚至输入(input)和输出(output))。

例如：

  • 新建一个SubGraph
  • 创建一个Boolean变量
  • 在另一个暴露出来的参数中的Visible if写下特定表达式关联之前的变量

参数中有两个名称Identifier和Label容易搞混。
判断语句要使用Identifier的命名，因为Identifier是真正的变量名(variable)
而Label是标签名，是外部展示的普通名称(是可以使用空格的)

看看效果：

Visible if输入框支持的语法和大部分编程语言的判断语句差不过：

  • 支持 布尔型(Boolean), 浮点型(Float), 整型(Integer)
  • true表示真，false表示假(不同的是，它这里首字符小写)
  • &&表示且，||表示或，！表示非
  • 比较操作：<,>, <=,>=, ==, !=

常用的判断例子：

Condition Formula
“myInput”是否为真 input["myInput"]
“myInput”是否为假 !input["myInput"]
“myParam”是否小于3 input[myParam] < 3
“myParam”是否大于等于3 input[myParam] >= 3
“myParam”的Z值是否等于3 input[myParam].z == 3
“myCondition0”和”myCondition1”都为真则真 input[myCondition0] && input[myCondition1]
